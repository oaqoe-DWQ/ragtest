# RAG评估指标原理详解

## 概述

本文档详细说明RAG评估系统中各个指标的含义、计算原理，以及它们如何反映知识库检索和chunk分块的质量。

---

## 一、数据结构

### 1.1 评估数据格式

每个评估样本包含以下核心字段：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `user_input` | 用户查询 | "如何设置闹钟？" |
| `retrieved_contexts` | 检索返回的chunk列表 | ["chunk1", "chunk2", "chunk3"] |
| `reference_contexts` | 标准答案的chunk列表 | ["标准chunk1", "标准chunk2"] |
| `response` | LLM生成的回答 | "设置闹钟的方法是..." |

### 1.2 Chunk分块概念

```
用户查询: "如何设置闹钟？"
    │
    ├── 检索返回的chunks (retrieved_contexts)
    │   ├── Chunk 1: "设置闹钟步骤第一步..." 
    │   ├── Chunk 2: "关于日历功能说明..."  ← 不相关
    │   └── Chunk 3: "修改闹钟时间的方法..."
    │
    └── 标准chunks (reference_contexts)
        ├── 标准Chunk A: "设置闹钟的具体步骤..."
        └── 标准Chunk B: "删除闹钟的操作方法..." ← 未召回
```

## 二、BM25评估指标

### 2.1 Context Precision（上下文精确率）

**定义**：检索到的chunk中，有多少是真正相关的。

**计算公式**：

```
Precision = 相关分块的语义得分总和 / 检索到的chunk总数
```

**示例**：

```
检索chunks: [Chunk1, Chunk2, Chunk3]
  Chunk1与标准ChunkA相似度: 0.9  → 相关
  Chunk2与标准ChunkB相似度: 0.2  → 不相关  
  Chunk3与标准ChunkA相似度: 0.8  → 相关

Precision = (0.9 + 0.8) / 3 = 0.567
```

**反映的问题**：
- **Precision低** → 检索返回了太多无关内容
- **可能原因**：分块太小导致噪音、分块策略不当、embedding模型问题

### 2.2 Context Recall（上下文召回率）

**定义**：标准答案中的chunk有多少被成功召回。

**计算公式**：

```
Recall = 相关分块的语义得分总和 / 标准chunk总数
```

**示例**：

```
标准chunks: [标准A, 标准B, 标准C]
  检索召回:
  - 标准A 被 Chunk1 匹配 (相似度0.9) ✓
  - 标准B 被 Chunk3 匹配 (相似度0.7) ✓
  - 标准C 没有被匹配 (相似度都<0.5) ✗

Recall = (0.9 + 0.7) / 3 = 0.533
```

**反映的问题**：
- **Recall低** → 知识库中缺少相关知识、分块粒度过粗/过细、embedding不准确

### 2.3 F1-Score

**定义**：Precision和Recall的调和平均。

**计算公式**：

```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

## 三、排序质量指标

### 3.1 MRR（Mean Reciprocal Rank，平均倒数排名）

**定义**：第一个相关chunk出现在检索结果中的平均位置。

**计算公式**：

```
MRR = (1/rank₁ + 1/rank₂ + ... + 1/rankₙ) / n

其中 rankᵢ = 第i个查询第一个相关chunk的位置
```

**示例**：

```
查询1: 相关chunk在位置3 → 1/3 = 0.333
查询2: 相关chunk在位置1 → 1/1 = 1.000
查询3: 无相关chunk → 0

MRR = (0.333 + 1.0 + 0) / 3 = 0.444
```

**反映的问题**：
- **MRR低** → 最重要/最相关的chunk没有被优先返回
- **可能原因**：排序算法问题、相关度计算不准确

### 3.2 MAP（Mean Average Precision，平均精度）

**定义**：所有查询的平均精度均值。

**核心概念**：

```
Precision@k = 前k个结果中相关chunk的数量 / k

AP = Σ Precision@k / 相关chunk总数
```

**示例**：

```
检索结果: [相关, 不相关, 相关, 相关]
  Precision@1 = 1/1 = 1.0
  Precision@3 = 2/3 = 0.667
  Precision@4 = 3/4 = 0.75
  
AP = (1.0 + 0.667 + 0.75) / 3 = 0.806
```

**反映的问题**：
- **MAP低** → 排序不够理想，相关内容被淹没在不相关内容中
- 关注的是**排序质量**，不仅要求相关，还要相关的内容排得靠前

### 3.3 NDCG（Normalized Discounted Cumulative Gain，归一化折损累积增益）

**定义**：考虑相关度等级的排序质量指标。

**计算公式**：

```
DCG = Σ (2^relᵢ - 1) / log₂(positionᵢ + 1)

IDCG = 理想排序下的DCG值

NDCG = DCG / IDCG
```

**示例**：

```
检索结果: [高相关, 不相关, 中相关, 不相关]
  位置1: 高相关(rel=1) → DCG贡献 = (2^1-1)/log₂(2) = 1/1 = 1.0
  位置2: 不相关(rel=0) → DCG贡献 = (2^0-1)/log₂(3) = 0
  位置3: 中相关(rel=1) → DCG贡献 = 1/log₂(4) = 0.5
  位置4: 不相关(rel=0) → DCG贡献 = 0

DCG = 1.0 + 0 + 0.5 + 0 = 1.5
IDCG = 理想排序[高,中,不,不] = 1.0 + 0.5 + 0 + 0 = 1.5
NDCG = 1.5 / 1.5 = 1.0
```

**反映的问题**：
- **NDCG低** → 高相关的内容被排在了后面
- 同时考虑**相关度等级**和**位置因素**

## 四、RAGAS评估指标

RAGAS是一套基于LLM的评估指标，需要调用API进行评估。

### 4.1 Context Precision（上下文精确率）

与BM25的Precision类似，但由LLM判断每个chunk是否对回答问题有贡献。

### 4.2 Context Recall（上下文召回率）

由LLM判断标准答案中的信息有多少被检索上下文覆盖。

### 4.3 Faithfulness（忠实度）

**定义**：LLM生成的回答是否忠实于检索到的上下文。

**计算**：
1. 从回答中提取所有陈述
2. LLM判断每个陈述是否可从上下文中推导
3. Faithfulness = 可推导陈述数 / 总陈述数

**示例**：

```
上下文: "苹果是一种水果，红色，甜的"
回答: "苹果是一种水果，产自山东"
陈述1: "苹果是一种水果" → ✓ 可推导
陈述2: "产自山东" → ✗ 上下文中没有

Faithfulness = 1/2 = 0.5
```

**反映的问题**：
- **Faithfulness低** → LLM产生了幻觉，回答中有上下文中没有的信息

### 4.4 Answer Relevancy（回答相关性）

**定义**：回答与用户问题的相关程度。

**计算**：
1. LLM根据回答生成多个假设问题
2. 计算原问题与假设问题的相似度
3. Answer Relevancy = 平均相似度

**反映的问题**：
- **Answer Relevancy低** → 回答偏离问题、没有正面回答问题

### 4.5 Context Entity Recall（上下文实体召回）

**定义**：标准答案中的实体有多少出现在检索上下文中。

**计算**：

```
Entity Recall = 上下文中出现的实体数 / 标准答案中的实体数
```

**反映的问题**：
- **Entity Recall低** → 检索遗漏了关键实体信息

### 4.6 Answer Correctness（回答正确性）

**定义**：回答的正确程度，综合考虑准确性和相关性。

**计算**：
1. LLM判断事实正确性
2. LLM判断语义相似度
3. 综合评分

## 五、指标与问题的映射关系

### 5.1 问题诊断表

| 指标表现 | 可能原因 | 优化方向 |
|----------|----------|----------|
| Precision低 | 分块太小、噪音多 | 调整分块策略、增大分块大小 |
| Recall低 | 知识库缺少内容、分块粒度问题 | 补充知识库内容、调整分块策略 |
| MRR低 | 相关chunk排位靠后 | 优化排序算法、调整相关性计算 |
| MAP低 | 排序不理想 | 同上 |
| NDCG低 | 高相关度内容排位靠后 | 同上 |
| Faithfulness低 | LLM幻觉 | 调低temperature、增加few-shot |
| Answer Relevancy低 | 回答跑题 | 检查prompt、调整LLM |

### 5.2 优化决策树

```
指标值异常
    │
    ├── Recall低 ←── 知识库内容问题？
    │   │           ├─ 是 → 补充知识库
    │   │           └─ 否 → 分块策略问题？
    │   │                   ├─ 粒度过细 → 合并chunk
    │   │                   └─ 粒度过粗 → 拆分chunk
    │   │
    │   └─ Precision低 ← 噪音chunk太多？
    │                   ├─ 是 → 增大分块、过滤噪音
    │                   └─ 否 → embedding质量问题
    │
    ├── MRR/MAP低 ←── 排序问题？
    │               ├─ 是 → 优化排序算法
    │               └─ 否 → 相关性计算问题
    │
    └── Faithfulness低 ← LLM幻觉？
                        └─ 调低temperature、增加上下文约束
```

## 六、相似度计算原理

### 6.1 语义相似度算法

系统使用综合相似度算法，结合多种文本匹配策略：

```python
def calculate_text_similarity(text1, text2):
    # 1. Jaccard相似度（词级别）
    jaccard = |words1 ∩ words2| / |words1 ∪ words2|
    
    # 2. 字符重叠度
    char_sim = |chars1 ∩ chars2| / |chars1 ∪ chars2|
    
    # 3. 子字符串匹配
    substring_sim = len(matching_substring) / len(longer_text)
    
    # 综合得分
    final = jaccard * 0.4 + char_sim * 0.3 + substring_sim * 0.3
    
    # 特殊处理：包含检测
    if shorter_text in longer_text:
        final = max(final, 0.8)  # 给予包含关系奖励
```

### 6.2 阈值配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `SIMILARITY_THRESHOLD` | 0.5 | 判断chunk是否相关的阈值 |
| `SEMANTIC_CONTAINMENT_THRESHOLD` | 0.9 | 语义包含度阈值 |

## 七、实际案例分析

### 7.1 检索优化案例

**问题**：Recall偏低 (0.6)

**分析**：
```
用户查询: "如何使用日历功能"
标准chunks: 5个
检索到的chunks: 3个
  - Chunk1 与标准A相似度0.9 ✓
  - Chunk2 与标准B相似度0.7 ✓
  - Chunk3 与标准C相似度0.1 ✗

未召回的标准chunk: D, E
```

**诊断**：知识库缺少D、E相关内容

**解决方案**：补充知识库内容

### 7.2 分块优化案例

**问题**：Precision偏低 (0.5)，大量irrelevant_chunks

**分析**：
```
用户查询: "预约会议的方法"
检索到的chunks:
  - "会议纪要格式模板..." (相似度0.2) ← 噪音chunk
  - "如何创建会议室..." (相似度0.8) ✓
  - "会议预约步骤如下..." (相似度0.9) ✓
```

**诊断**：分块太小，导致无关内容混入

**解决方案**：增大分块大小，减少碎片化

## 八、总结

| 指标类型 | 关注维度 | 优化方向 |
|----------|----------|----------|
| Precision | 检索准确性 | 分块策略、去噪 |
| Recall | 检索完整性 | 知识库完善、分块粒度 |
| MRR | 第一相关结果位置 | 排序算法 |
| MAP | 整体排序质量 | 排序算法 |
| NDCG | 考虑相关度等级的排序 | 排序算法 |
| Faithfulness | 回答真实性 | LLM配置、prompt |
| Answer Relevancy | 回答相关性 | prompt优化 |
