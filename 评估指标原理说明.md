# RAGAS 评估指标原理详解

## 一、概述

RAGAS 是一套基于 LLM 的 RAG 系统评估框架。本系统使用 RAGAS v0.3.2，支持 8 个评估指标，其中 `ContextEntityRecall` 使用自定义稳定实现（基于规则，不依赖 LLM），其余 7 个依赖 LLM 或 Embedding。

---

## 二、输入输出格式

RAGAS 评估以 **样本（Sample）** 为基本单位，每个样本的结构：

| 字段 | 说明 |
|------|------|
| `user_input` | 用户问题 |
| `retrieved_contexts` | 检索返回的 chunks（列表） |
| `reference` | 标准参考答案（ground truth answer） |
| `reference_contexts` | 标准答案对应的 chunks（列表） |
| `response` | LLM 生成的回答 |

---

## 三、8 个评估指标

### 3.1 Faithfulness（忠实度）

**依赖**：LLM

**输入**：`user_input` + `retrieved_contexts` + `response`

**输出**：0 ~ 1 的分数

**原理**：将 `response` 拆分为多个陈述，LLM 判断每个陈述能否从 `retrieved_contexts` 中推断出来。分数 = 可推断陈述数 / 总陈述数。

**低分含义**：回答中有上下文中不存在的信息（幻觉）。

---

### 3.2 Answer Relevancy（回答相关性）

**依赖**：LLM + Embedding

**输入**：`user_input` + `response`

**输出**：0 ~ 1 的分数

**原理**：LLM 根据 `response` 生成多个假设问题，计算这些假设问题与原始 `user_input` 的 embedding 余弦相似度，取平均。

**低分含义**：回答没有正面回应问题，或回答内容偏离问题。

---

### 3.3 Context Precision（上下文精确度）

**依赖**：LLM

**输入**：`user_input` + `retrieved_contexts` + `reference`

**输出**：0 ~ 1 的分数

**原理**：LLM 对 `retrieved_contexts` 中的每个 chunk 按顺序判断其对回答 `reference` 是否有贡献，按位置加权（越靠前权重越高），计算加权精确度。

**低分含义**：检索结果中混杂了太多无关 chunk，或相关 chunk 排位靠后。

---

### 3.4 Context Recall（上下文召回率）

**依赖**：LLM（本系统使用中文提示词版本 ChineseContextRecall）

**输入**：`retrieved_contexts` + `reference_contexts`

**输出**：0 ~ 1 的分数

**原理**：将 `reference_contexts` 拆分为多个陈述，LLM 判断每个陈述能否从 `retrieved_contexts` 中推断出来。分数 = 可推断陈述数 / 总陈述数。

**低分含义**：检索到的内容没有覆盖标准答案的关键信息。

---

### 3.5 Context Entity Recall（上下文实体召回率）

**依赖**：规则（正则表达式），**不需要 LLM**

**输入**：`retrieved_contexts` + `reference_contexts`

**输出**：0 ~ 1 的分数

**原理**：从 `reference_contexts` 中用正则提取实体（产品名、数字、日期、专业术语等），再从 `retrieved_contexts` 中提取实体，计算召回率 = 上下文中出现的标准实体数 / 标准实体总数。

**低分含义**：检索遗漏了关键实体（如数字、时间、专有名词等）。

---

### 3.6 Context Relevance（上下文相关性）

**依赖**：LLM（本系统使用中文提示词版本 ChineseContextRelevance）

**输入**：`user_input` + `retrieved_contexts`

**输出**：0 ~ 1 的分数

**原理**：LLM 评估 `retrieved_contexts` 整体与 `user_input` 的相关性，返回 0 / 1 / 2 的评分，映射到 0 ~ 1 范围。

**低分含义**：检索返回的上下文大部分与问题无关。

---

### 3.7 Answer Correctness（回答正确性）

**依赖**：LLM + Embedding

**输入**：`user_input` + `response` + `reference`

**输出**：0 ~ 1 的分数

**原理**：LLM 综合评估 `response` 与 `reference` 的事实正确性和语义相似度，权重可配置。综合两个维度给出最终分数。

**低分含义**：回答与标准答案偏差较大，可能包含错误或遗漏关键信息。

---

### 3.8 Answer Similarity（回答相似度）

**依赖**：Embedding（**不需要 LLM**）

**输入**：`response` + `reference`

**输出**：0 ~ 1 的分数

**原理**：使用 embedding 模型分别编码 `response` 和 `reference`，计算余弦相似度。

**低分含义**：回答与参考答案在语义层面差异较大。

---

## 四、指标依赖汇总

| 指标 | 依赖 | 说明 |
|------|------|------|
| Faithfulness | LLM | 幻觉检测 |
| Answer Relevancy | LLM + Embedding | 回答是否切题 |
| Context Precision | LLM | 检索精确度 + 排序 |
| Context Recall | LLM | 检索召回能力 |
| Context Entity Recall | 规则 | 实体信息遗漏检测 |
| Context Relevance | LLM | 检索内容整体相关性 |
| Answer Correctness | LLM + Embedding | 回答综合正确性 |
| Answer Similarity | Embedding | 回答语义相似度 |

---

## 五、问题诊断参考

| 指标表现 | 可能原因 |
|----------|----------|
| Context Precision 低 | 检索噪音多、分块粒度过细 |
| Context Recall 低 | 知识库缺少内容、分块切分了关键信息 |
| Context Relevance 低 | 检索结果整体偏题 |
| Context Entity Recall 低 | 遗漏关键数字/专有名词 |
| Faithfulness 低 | LLM 幻觉，temperature 过高 |
| Answer Relevancy 低 | 上下文信息不足或 prompt 引导不足 |
| Answer Correctness 低 | 综合问题，需同时排查检索和生成 |
| Answer Similarity 低 | 回答措辞/表达与参考答案差异大 |
